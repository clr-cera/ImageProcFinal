# Classical Cup Detection

We made three classical (non–deep-learning) pipelines for detecting cups in 256×256 images labelled in YOLO format. All share a linear-SVM classifier and the same seeded 5-fold cross-validation split, and differ mainly in how they generate the candidate boxes the classifier scores (and, for the last one, in the descriptor). Each script prints a metrics table (precision/recall/F1/accuracy swept over score thresholds), writes it to `metrics.txt`, and saves annotated images.

`hog_svm.py` describes each window with a Histogram of Oriented Gradients vector, trains a linear SVM on ground-truth cup crops (plus horizontal flips) against random negatives, then mines hard negatives (high-scoring windows that miss every cup) and retrains. At test time it scans an image pyramid with a sliding window, scores every position and scale, and merges overlapping boxes with non-maximum suppression. Because the sliding window yields thousands of mostly-background candidates per image, precision is low. Output goes to `detections/` (green = ground truth, red = detection).

`hough_experiment.py` exploits cup geometry: a cup's circular opening projects to an ellipse, so the rim is a strong elliptical edge. It reuses the trained HOG+SVM but generates candidates differently — it downscales each image, runs Canny edge detection and `hough_ellipse` to propose rims, and grows each rim downward into a cup box. The SVM then verifies every proposal, keeping Hough's high recall while rejecting the many false rim-edges (plates, shadows, arcs). It also mines extra hard negatives from the Hough proposals the SVM wrongly accepts on the training set (capped at `MAX_HOUGH_NEG` since they are highly redundant). Output goes to `hough_detections/` (green = ground truth, blue = proposed rim, red = accepted cup box).

`felze_correlogram_hog.py` finds objects by segmentation instead of geometry: it runs the Felzenszwalb–Huttenlocher graph-based algorithm (`felzenszwalb`) at several scales and treats each segment's bounding box as a candidate object. Because a segment groups pixels but says nothing about *what* they are, a linear SVM accepts or rejects each box, and here the descriptor is richer than plain HOG — it concatenates the grayscale HOG vector (cup silhouette / rim edges) with an RGB **color autocorrelogram** (64 quantized colors × 4 Chebyshev distances, capturing the spatial color texture of porcelain, handles, and drinks). It mines hard negatives from the segmentation proposals the SVM wrongly accepts on the training set (capped at `MAX_FELZ_NEG`) and retrains. Output goes to `felze_detections/` (green = ground truth, blue = proposed segment box, red = accepted cup box).

**Results** (5-fold cross-validation pooled over all 30 images / 83 ground-truth cups, so the figures are stable across seeds; best-F1 operating point, full sweep in `metrics.txt`): the plain HOG sliding window peaks at only P≈0.06 / R≈0.07 / F1≈0.06, while the Hough-proposal + HOG pipeline roughly doubles that to P≈0.16 / R≈0.11 / F1≈0.13 (at score threshold 0.25), and its precision keeps climbing to ≈0.27 at the strictest cutoff. The Felzenszwalb-proposal + correlogram/HOG pipeline lands in between, peaking at P≈0.12 / R≈0.20 / F1≈0.15 (at score threshold 0.0); segmentation recovers more candidates than sliding windows but its boxes often cut a cup into rim/body/handle pieces or merge it with the background, so box-IoU and recall suffer despite the richer color descriptor. The geometry-guided Hough proposals still win on precision — the SVM sees far fewer, more cup-like candidates. These pooled numbers are much lower than earlier single-split runs (F1 up to 0.53), which were optimistic noise on a handful of lucky test images; on a dataset this small (30 images), absolute scores stay low and only the relative ordering is meaningful.

Run with `uv run hog_svm.py`, `uv run hough_experiment.py`, or `uv run felze_correlogram_hog.py`.
